# 验机侦探 深度检测系统设计文档

> 版本：v1.1 | 更新日期：2026-06-01 | 状态：已修订（v1.0→v1.1）

## 概述

### 项目定位

验机侦探是一款软硬件结合的设备检测工具，通过"小程序免费引导 + 电脑端深度检测"的混合架构，为用户提供 iPhone/iPad/Mac 设备的全面验机服务。

### 核心价值

| 价值 | 说明 |
|------|------|
| **信任背书** | 两元店认证，报告具有公信力 |
| **真实数据** | 电脑端数据线直连，获取硬件真实参数 |
| **安全透明** | 用户授权后才读取，数据不上传云端 |
| **通行证协同** | 小程序+电脑端互通，2元买断制 |

### 技术架构总览

### 1.4 报告设计

#### H5报告内容

| 区块 | 内容 | 数据来源 |
|------|------|---------|
| 报告头部 | 两元店认证标识 + 报告编号 | 系统生成 |
| 设备信息 | 型号/序列号/IMEI/颜色/存储 | **电脑端读取（不可伪造）** |
| 外观评分 | 1-10分 + 描述 | 小程序端手动检测 |
| 功能评分 | 各模块评分 | 小程序端手动检测 |
| 硬件参数 | 电池循环/激活锁状态等 | **电脑端读取（不可伪造）** |
| 拍照记录 | 用户上传的外观照片 | 小程序端 |
| 检测时间 | 报告生成时间 | 系统生成 |
| 水印 | 两元店认证 + 用户ID + 时间戳 | 系统生成 |
| 验证入口 | 二维码（扫码验证报告真伪） | 系统生成 |

**重要：所有关键硬件数据（序列号/IMEI/电池信息）必须来自电脑端数据线读取，小程序端不提供手动录入入口。**

#### 水印规范

| 元素 | 位置 | 样式 |
|------|------|------|
| 两元店Logo | 四角 | 透明度30%，白色/黑色自适应 |
| 用户ID | 底部 | 小字，透明度20% |
| 时间戳 | 底部 | 小字，透明度20% |

## 第二章：电脑端设计

### 2.1 功能定位

电脑端作为**深度检测核心**，通过数据线连接设备获取真实硬件参数，并提供完整的引导验机流程。

### 2.2 支持平台与数据来源

| 设备 | 平台 | 数据来源 | 关键技术 |
|------|------|---------|---------|
| iPhone/iPad | Windows | libimobiledevice | ideviceinfo/idevice_id/idevicebackup |
| iPhone/iPad | macOS | libimobiledevice + IOKit | 同上 + IORegistry |
| Mac | macOS | system_profiler + IOKit | system_profiler/SPHardwareDataType |
| Mac | Windows | N/A（初期不支持） | - |

### 2.3 深度检测项目

#### iOS 设备（libimobiledevice 获取）

| 参数 | 说明 | 获取命令 |
|------|------|---------|
| 设备型号 | Model Number | `ideviceinfo -k ModelNumber` |
| 设备名称 | Device Name | `ideviceinfo -k DeviceName` |
| 序列号 | Serial Number | `ideviceinfo -k SerialNumber` |
| IMEI | IMEI | `ideviceinfo -k InternationalMobileEquipmentIdentity` |
| 激活锁状态 | Find My iPhone | `ideviceinfo -k FindMyPhoneEnabled` |
| 电池序列号 | Battery Serial | `ideviceinfo -k BatterySerialNumber` |
| 电池循环次数 | Battery CycleCount | `ideviceinfo -k BatteryCycleCount` |
| 零件维修历史 | Repair History | iOS 17+ CHRepairability Suitability API |
| 设备颜色 | Color | `ideviceinfo -k DeviceColor` |
| 存储容量 | StorageCapacity | `ideviceinfo -k StorageCapacity` |
| 固件版本 | Firmware Version | `ideviceinfo -k FirmwareVersion` |

#### iOS 设备（IOKit/iOS 17+ API 获取）

| 参数 | 说明 | 获取方式 | 支持情况 |
|------|------|---------|---------|
| 电池实际容量 | DesignCapacity vs ActualCapacity | IOKit | 部分机型支持 |
| 零件更换状态 | 屏幕/电池/摄像头更换记录 | CHRepairability API | **仅部分机型支持，需降级方案** |
| 生产日期 | 电池/零件生产日期 | 零件序列号解析 | 部分支持 |

**注意：CHRepairability API仅支持iOS 17+的部分机型，且需要设备已连接并完成配对授权。对于不支持的机型，报告应显示"零件维修历史：无法获取"而非空白或错误。**

#### Mac 设备（system_profiler + IOKit）

| 参数 | 说明 | 获取命令 |
|------|------|---------|
| 型号名称 | Model Name | `system_profiler SPHardwareDataType` |
| 序列号 | Serial Number | 同上 |
| 处理器 | CPU | 同上 |
| 内存 | RAM | 同上 |
| 存储容量 | Storage | 同上 |
| 电池循环次数 | Battery Cycle Count | `system_profiler SPBatteryDataType` |
| 电池健康度 | Battery Health | 同上 |
| 零件维修历史 | AppleCare 状态 | `system_profiler SPRepairDataType` |

## 第三章：通行证系统

### 3.1 设计原则

- 2元买断制，终身有效
- 小程序与电脑端通行证互通
- 通行证实录存储于云端（微信云开发或自建后端）

### 3.2 通行证数据结构

```json
{
  "passport_id": "uuid",
  "user_id": "微信UnionID或设备指纹",
  "platform": "miniprogram|windows|macos",
  "purchased_apps": ["验机侦探"],
  "purchase_time": "2026-06-01",
  "expire_time": null,
  "status": "active"
}
```

### 3.3 验证流程

| 场景 | 验证方式 |
|------|---------|
| 小程序内 | 检查微信云开发数据库中UnionID对应通行证 |
| 电脑端首次使用 | 要求用户扫码登录小程序授权，获取UnionID |
| 电脑端后续使用 | 本地缓存通行证状态，定期同步验证 |

### 3.4 电脑端登录方案

| 方案 | 说明 |
|------|------|
| 扫码登录（推荐） | 电脑端显示二维码，用户小程序内扫码授权 |
| 账号密码 | 用户注册账号（增加注册门槛，不推荐） |

**扫码登录流程：**
```
电脑端启动
    │
    ▼
显示二维码（包含临时token）
    │
    ▼
用户小程序内扫一扫
    │
    ▼
小程序调用云函数验证token
    │
    ▼
云函数返回UnionID + 通行证状态
    │
    ▼
电脑端缓存通行证，本地解锁高级功能
```

### 4.1 报告类型

| 类型 | 内容 | 权限 |
|------|------|------|
| 基础报告 | 手动检测结果 | 免费（小程序/电脑端均可生成） |

```
https://report.example.com/verify/{report_id}?token={signature}
```

- `report_id`: 报告唯一标识
- `signature`: 服务端签名，防止伪造

#### 报告页面结构

### 4.3 二维码验证

- 每个报告有唯一二维码
- 扫码进入验证页面
- 验证页面仅显示：设备型号 + 报告时间 + 真伪状态
- 不显示完整报告内容（防止隐私泄露）

| 功能 | 说明 |
|------|------|
| 格式 | PDF/A（长期保存格式） |

## 第五章：安全与合规

### 5.1 数据安全

| 原则 | 实现方式 |
|------|---------|
| 本地处理 | 硬件数据仅在用户电脑本地处理 |
| 不上传云端 | 检测数据不发送至服务器（除通行证验证外） |
| 传输加密 | 所有网络通信使用HTTPS |
| 报告签名 | 报告Token使用HMAC-SHA256防伪 |

### 5.2 用户隐私

| 承诺 | 说明 |
|------|------|
| 最小采集 | 仅采集验机必需数据 |
| 本地存储 | 报告数据存储在用户本地 |
| 删除权 | 用户可请求删除云端通行证记录 |

### 5.3 微信平台合规

| 要求 | 实现方式 |
|------|---------|
| 不强制获取手机号 | 通行证与UnionID绑定，无需手机号 |
| 隐私政策 | 明确告知数据收集范围 |
| 儿童保护 | 16岁以下用户需监护人同意 |

```
    │
```
```
    ▼
```
